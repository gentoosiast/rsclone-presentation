tech stack

Nest.js

  • good modular architecture out of the box
  • dependency injection pattern (inspired by Angular)
  • first-class TypeScript support
  • dozens of built-in modules that help you easily integrate with common technologies and concepts
  • higher learning curve in comparison to Express

Socket.IO

  • disconnect detection and automatic reconnect
  • rooms support out of the box
  • first class support of TypeScript
  • very good official documentation
  • none (for our use case)

TypeORM

  • easy integration with Nest.js
  • it can be used with both relational databases, and NoSQL like MongoDB
  • good documentation
  • none (for our use case)

SQLite

  • easy to integrate into the project (just npm install)
  • doesn't require much configuration
  • lacks many features of more advanced databases

React

  • powerful and versatile
  • huge community
  • lots of 3rd party components for every use case
  • none (for our use case)

Библиотека для создания пользовательских интерфейсов React

Zustand

  • very lightweight
  • can be used with other frameworks or vanilla JavaScript
  • no need for any context wrapper
  • documentation doesn’t cover all of the use cases

  • good community
  • lots of examples on official site
  • good official documentation

Yandex version / Our version

Когда мы определились,что будем делать игру, то стали изучать варианты дизайна. Вариант яндекса нам показался слишком простым. Мы нашли более интересный вариант реализации мобильной игры, и решили сделать десктопный аналог. Приняли решение использовать Фейзер, чтобы изучить новый для нас фреймворк.

Game scene

Сейчас мы покажем, что из этого вышло. Если пользователь создает комнату, он становится хостом, и ожидает других игроков. Для него отрисовывается пустой стол. Кнопка старт пока не активна.

Game scene

2ой игрок входит в эту же комнату, для него также отрисовался стол. После входа 2го игрока у хоста активируется кнопка - начать игру.

Game scene

Здесь видим, как выглядит комната для пользователя, который не является хостом. У него нет кнопки старт и есть надпись, предлагающая ожидать начала игры.

the game starts

после начала игры происходит раздача карт игрокам из колоды и распределение ролей - атакующий (выделен красным), защищающийся (выделен зеленым) и ожидающие. У активного игрока рамка с таймером.

the game is on

Если защищающийся берет карты, то напротив его иконки появляется всплывающее сообщение, а так же подсказка справа над столом текущего игрока.

the game is on

У активного игрока, если он защищающийся, подсвечиваются карты, которыми можно отбить. Также он может взять карты - у него появляется соответствующая кнопка.

the game is on

Если активный игрок - нападающий, то у него подсвечиваются карты, которые можно подкинуть. Или он может нажать пасс.

the game is on

В каждой комнате есть чат. Также, как видно на слайде, после раздачи карты игроков сортируюся по мастям, козырные карты в конце

the game is on

Если чат свернут, то приходят звуковые и визуальные уведомления о сообщениях. В нашем случае 1 непрочитанное сообщение.

the game is on

Если на столе больше 3х пар карт, то карты располагаются в 2 ряда.

the game is on

Когда 1 из игроков становится победителем, в нашем случае гентузиаст, то он может остаться в комнате в качестве наблюдателя. Если бы в игре было 2 игрока, то игра бы закончилась, но в нашем примере игроков 3, поэтому игра продолжается между игроками с картами.

the game is on

Победивший игрок также может выйти из комнаты, при этом игра не закончится. Здесь показано, как выглядит комната для победителя - у него появляется попап.

the game ends

Когда остается только 1 игрок с картами, он становится дураком и игра завершается. У всех игроков, не являющихся хостами, появляется такой попап.

the game ends

У хоста же после окончания игры есть возможность открыть комнату, чтобы присоединились новые игроки, или перезапустить игру с текущим составом игроков.

the game ends

Если все игроки, кроме него выходят из комнаты (мы видим у них значок офлайн), тогда кнопка старт, естественно, становится неактивной, и хост может только открыть комнату

the game ends

Если же раньше выходит хост, то хостом становится другой игрок. Видим у игрока попап, что теперь он является хостом.
